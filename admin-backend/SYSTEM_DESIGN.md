# 静莱美代理商后台管理系统 - 架构设计

## 系统概述
静莱美代理商后台管理系统是一个为管理员和超级用户设计的完整管理平台，用于管理小程序前端的所有业务逻辑、数据统计和内容管理。

## 核心设计理念：**板块逻辑串联**

所有功能模块不是孤立的，而是通过数据和业务流程相互关联，形成一个有机的整体。

## 一、系统架构

### 1. 前端架构
- **技术栈**：React + TypeScript + Ant Design + Recharts
- **状态管理**：Redux + React Query
- **路由**：React Router v6
- **UI框架**：Ant Design Pro

### 2. 后端架构
- **API服务**：Node.js + Express
- **数据库**：MySQL + Redis缓存
- **认证授权**：JWT + RBAC角色权限
- **文件存储**：阿里云OSS/MinIO

### 3. 数据流架构
```
小程序用户 → 前端API → 后台管理系统 → 数据分析 → 决策支持
```

## 二、核心功能模块及关联关系

### 模块1：**用户管理模块**
- **主要功能**：代理商注册审核、账户状态管理、权限分配
- **关联模块**：
  - → 团队管理（上级-下级关系）
  - → 收益统计（个人业绩）
  - → 订单管理（购买记录）
  - → 商学院（学习进度）

### 模块2：**商品管理模块**
- **主要功能**：商品上下架、价格设置、库存管理
- **关联模块**：
  - ← 用户管理（代理价格等级）
  - → 订单管理（商品销售）
  - → 收益统计（商品销售额）
  - → 库存预警

### 模块3：**订单管理模块**
- **主要功能**：订单审核、发货管理、退款处理
- **关联模块**：
  - ← 用户管理（订单归属）
  - ← 商品管理（商品信息）
  - → 收益统计（订单金额）
  - → 团队管理（团队业绩）

### 模块4：**收益统计模块**
- **主要功能**：佣金计算、提现审核、业绩报表
- **关联模块**：
  - ← 用户管理（收益归属）
  - ← 订单管理（订单来源）
  - ← 团队管理（团队提成）
  - → 数据分析（趋势预测）

### 模块5：**团队管理模块**
- **主要功能**：团队结构、成员管理、业绩排名
- **关联模块**：
  - ← 用户管理（团队成员）
  - ← 收益统计（团队业绩）
  - → 商学院（团队培训）

### 模块6：**商学院管理模块**
- **主要功能**：内容发布、学习进度、话术库
- **关联模块**：
  - ← 用户管理（学员信息）
  - → 团队管理（团队学习）
  - → 数据分析（学习效果）

### 模块7：**数据分析看板**
- **主要功能**：实时数据、趋势分析、智能预警
- **关联模块**：所有模块数据汇总分析

## 四、核心业务流程

### 1. **代理商注册到收益提现全流程**
```
新代理商注册 → 资质审核 → 账户激活 → 购买商品 → 产生订单 → 
订单完成 → 佣金计算 → 收益统计 → 申请提现 → 提现审核 → 
资金到账 → 数据分析
```

### 2. **商品销售到库存管理流程**
```
商品发布 → 价格设置 → 代理商购买 → 订单生成 → 库存扣减 → 
库存预警 → 补货通知 → 物流发货 → 确认收货 → 售后处理
```

### 3. **团队发展与培训流程**
```
发展下级 → 团队建立 → 业绩考核 → 培训需求 → 内容发布 → 
学习跟踪 → 效果评估 → 优化培训 → 团队成长 → 业绩提升
```

## 六、数据库关联设计

### 核心表关系
1. **用户表** (users) ↔ **团队表** (teams) - 一对多/多对多
2. **用户表** (users) ↔ **订单表** (orders) - 一对多
3. **商品表** (products) ↔ **订单明细** (order_items) - 一对多
4. **订单表** (orders) ↔ **收益表** (commissions) - 一对多
5. **用户表** (users) ↔ **学习进度** (study_progress) - 一对多

### 关键关联字段
- `user.parent_id` - 上级代理ID
- `order.user_id` - 订单所属用户
- `commission.order_id` - 收益来源订单
- `study_progress.user_id` - 学习用户ID
- `team.leader_id` - 团队负责人ID

## 八、权限控制模型

### RBAC角色权限
1. **超级管理员** - 所有权限
2. **运营管理员** - 用户管理、订单管理、内容管理
3. **财务管理员** - 收益管理、提现审核、财务报表
4. **内容管理员** - 商学院内容管理
5. **客服专员** - 用户咨询、订单处理

### 数据权限控制
- 部门数据隔离
- 团队数据可见性
- 个人数据隐私保护

## 九、监控与预警系统

### 监控指标
1. **业务监控**：订单量、销售额、用户增长
2. **系统监控**：API响应时间、数据库性能、服务器负载
3. **安全监控**：异常登录、数据篡改、权限变更

### 预警规则
- 库存低于阈值
- 收益异常波动
- 团队业绩下滑
- 学习进度停滞
