# 静莱美代理商后台管理系统 - 系统访问与使用指南

## 📖 系统简介

静莱美代理商后台管理系统是一个现代化的React应用，为代理商提供全方位的业务管理功能，包括用户管理、商品管理、订单管理、收益管理、团队管理、商学院等功能。

## 🚀 快速启动

### 方式一：使用启动脚本（推荐）

```bash
# 进入项目目录
cd /Users/apple/WorkBuddy/20260324191412

# 启动React开发服务器
./start_react_server.sh

# 或者使用简化版启动脚本
```

### 方式二：手动启动

```bash

# 设置Node环境变量
export NODE_PATH="/Users/apple/.workbuddy/binaries/node/workspace/node_modules"

# 启动服务器
/Users/apple/.workbuddy/binaries/node/versions/20.18.0/bin/node node_modules/.bin/react-scripts start
```

### 方式三：使用Node.js直接启动

```bash

npm start
# 或者
yarn start
```

## 🌐 访问系统

1. **打开浏览器**，访问：`http://localhost:3000`
2. **系统会自动重定向**到登录页面：`http://localhost:3000/login`
3. **登录凭据**：
   - **用户名**：任意3个或以上字符（如：admin, test, user等）
   - **密码**：任意6个或以上字符（如：password, 123456等）
4. **登录成功后**：自动跳转到仪表板页面

## 🔧 技术架构

### 前端技术栈
- **框架**: React 18 + TypeScript
- **UI库**: Ant Design 5.0
- **状态管理**: Redux Toolkit + Zustand
- **路由**: React Router 6
- **HTTP客户端**: Axios
- **图表**: Ant Design Charts + Recharts

### 项目结构
```
admin-backend/src/
├── api/                    # API接口定义
├── components/            # 通用组件
├── pages/                 # 页面组件
├── store/                # 状态管理（Redux）
├── services/             # 数据服务层
├── types/                # TypeScript类型定义
├── utils/                # 工具函数
├── styles/               # 样式文件
└── test/                 # 测试文件
```

## ⚠️ 注意事项

### 1. 浏览器兼容性
- 推荐使用 Chrome 90+、Firefox 88+、Edge 90+
- 不支持IE浏览器

### 2. 屏幕分辨率
- 推荐分辨率：1920×1080 或更高
- 支持响应式设计，适配移动端

### 3. 网络要求
- 需要稳定的网络连接
- API响应依赖于后端服务状态

### 4. 安全建议
- 定期更改管理员密码
- 设置访问IP白名单
- 启用HTTPS加密传输
- 定期备份系统数据

## 📞 技术支持

### 问题排查
1. **浏览器控制台**: 查看JavaScript错误信息
2. **网络面板**: 检查API请求和响应
3. **React开发者工具**: 检查组件状态和性能
4. **Redux开发者工具**: 检查状态管理

### 常见问题
1. **无法访问页面**: 检查服务器是否启动，端口是否被占用
2. **登录失败**: 检查用户名和密码格式，查看API响应
3. **页面加载慢**: 检查网络连接，查看控制台错误信息
4. **数据不显示**: 检查API服务状态，查看Redux状态

## 📋 系统状态检查

### 健康检查
```bash
# 检查服务器是否运行
curl -I http://localhost:3000

# 检查API响应
curl http://localhost:3000/api/health

# 检查编译状态
npm run build
```

### 性能监控
- **页面加载时间**: 应在2秒以内
- **API响应时间**: 应在100ms以内
- **内存使用**: 监控Chrome内存使用情况
- **CPU使用率**: 确保服务器资源充足
